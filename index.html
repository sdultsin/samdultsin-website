How I Work

When I needed a website, I didn't wait to "learn web development properly." I spent a weekend figuring out what tools existed (Cursor, Claude, GitHub, Vercel), built it, deployed it, and moved on.

Any problem that can be solved with more information can be solved with a human + agents.
It's insane. Yet people still try to solve problems manually.

1

Start with what you want to build, not the basics

2

Learn only what's blocking you right now

3

Build something broken

4

Iterate until it's no longer broken

From my experience, this top-down learning process has been 10x more efficient than the typical "start with the basics" format that college seems to teach.

On College

I left college for a simple reason: it’s an inefficient way to learn.

Theory ≠ learning

  • You don't learn how to ride a bike by studying sensorimotor mechanics
  • You don't learn multi-billion dollar M&As from a FIN 435 class
  • You don't learn PMF from a case study about Airbnb

There are startups everywhere looking for talent. Real companies with real problems. College just didn't make sense when I could be doing the actual work instead of preparing for it.
